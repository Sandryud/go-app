# Health API

## Общие сведения

- **Базовый URL**: `http://localhost:8080`
- **Версия API**: без префикса (`/health`, `/health/db`)

### Формат ошибок

Health‑эндпоинты обычно возвращают только статус `200 OK` или системные ошибки
в формате по умолчанию Gin. Для диагностики инфраструктуры этого достаточно.

---

## Эндпоинты

### GET `/health`

- **Описание**: базовый health‑check сервера.
- **Назначение**: использовать в балансировщиках, оркестраторах (k8s livenessProbe),
  а также для быстрой ручной проверки «жив ли процесс сервера».
- **Ответы**:
  - `200 OK` — сервер жив и основной цикл обработки запросов работает.

Пример запроса:

```bash
curl -i http://localhost:8080/health
```

---

### GET `/health/db`

- **Описание**: проверка доступности базы данных.
- **Назначение**: использовать для monitoring/alerting и readiness‑проверок (k8s readinessProbe),
  чтобы убедиться, что приложение может работать с БД.
- **Ответы**:
  - `200 OK` — подключение к БД успешно, запрос `SELECT 1` выполняется без ошибок.

Пример запроса:

```bash
curl -i http://localhost:8080/health/db
```
